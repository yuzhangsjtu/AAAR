# AI 作为独立行动者(Agent Level)

五级角色框架行至第四级,Agent Level 与前三级之间存在一道被低估的分水岭。RA、Supervisor、Domain Expert 三级都把 AI 摆在研究的外部位置:它做工、它陪想、它代为判断,但研究本身的因果叙事里没有它的位置。Agent Level 把这道边界向内推一格,LLM 出现在研究的因果链内部,担任一个原本由真人占据的位置,要么作为被试,要么作为决策者,要么作为研究者把整段流程交给它跑。论文的方法论描述、伦理审查的对象、复现性的判断标准、同行评议的关注点,都因此发生改写。

围绕这一改写的争议,在 2024 至 2026 年间从计算社会科学外溢到经济学、心理学、自然科学,几乎每个有数据传统的学科都在重新书写自己关于「研究中的 actor 究竟是谁」的定义。本章不试图给出一个学科性的定论,而是建立一组判别工具:三种 LLM 身份的判定法、硅样本研究的论文写作底线、以及「行动者」概念在 LLM 时代的边界。沿着这三条线,一个研究者可以判断自己即将开展的工作是合规的方法学探索,还是已经滑入了被流畅叙述掩盖的科学过度延伸。


## 角色升级:从工具使用到本体融入

### 五级框架第四级的质变

第十一章把 AI 作为研究助理的边界讲清楚 [@sec-silicon-sample-as-method 已点出硅样本作为方法论假设];第十二、十三章分别处理对话式导师与领域专家的判断委托。这三级共有一个隐含前提:研究的故事可以在不提到 AI 的情况下被完整讲完,AI 只是在背后让流程运转得更快、判断得更广。Agent Level 的不同在于,从这一级开始,AI 出现在论文的方法部分,出现在数据生成的源头,出现在被推论对象的位置上;移除它,研究本身就不存在了。

OpenAI 给出的五级框架把 Agent 描述为一个能自主完成跨步骤任务的系统,第一章已给出对应的工程定义。Agent Level 在本书脉络里把这一定义向方法论方向再扩一层:工程意义上的自主性是 Agent Level 的必要条件,而非充分条件;在本章意义上构成 Agent Level 的,是 AI 的输出在研究的因果故事里被解释为一个真人式 actor 的输出 [@bommasani2022opportunities]。这种身份滑移在第四章关于「执行性 vs 思考性任务」的讨论(line 130-136)已被预警:当自动化系统的输出被放入研究者本应承担的位置,执行与思考的边界会被悄悄踩过,而踩过的代价直到论文进入同行评议甚至刊发之后才显现。

一个具体的例子能凸显两种用法的差别。同一台 GPT-4o,如果用来从两百篇论文中提取「研究问题、数据、方法、发现」四要素,这是 RA Level;如果用来在博弈实验中扮演 1000 个匿名被试,这是 Agent Level。同一台模型、同一段 prompt 模板、甚至同一段返回文本,在两种用法里承担的方法论角色完全不同。Dell'Acqua 等人在 2023 年的现场实验已显示,把工具用法与代理用法混在一起,会让任务效果出现「锯齿状」分布,熟练用户与新手在同一种工具下产生方向相反的产出 [@dellacqua2023navigating]。这一实证结果提示:即便底层技术相同,方法论身份的不同会在实操层面造成可被量化的效果差异。

### 「行动者」概念在社会科学传统中的承诺

要理解为什么把 LLM 称为 actor 在方法论上具有特殊重量,需要回到「行动者」一词在社会科学传统里的承诺。韦伯把社会行动定义为带有主观意义的行动,行动者的概念由此承担三件事:意图、对情境的解释、对结果的承担 [@TODO-weber-economy-society-1922]。经济学的 rational agent 把这套承诺改写为效用函数加约束加选择,但「选择」一词背后仍假定有一个能在多种可能间分配权重的主体。心理学的 person 概念则要求可重复观察的内部状态:同一被试在两次测验中表现出来的差异,需要被解释为状态变化或情境差异,而不是数据噪声。

LLM 的输出与上述任何一种 actor 概念都不严格对应。它产生输出的方式更接近一个条件概率分布上的采样过程:给定 prompt,从训练数据塑造出的潜在分布中抽出一段文本。Bender 等人 2021 年那篇被反复引用的「随机鹦鹉」论文已指出,这种采样过程不必然包含对意义的理解 [@bender2021dangers]。如果 actor 概念预设了「对情境的解释」,那么把 LLM 直接放进 actor 的位置就需要在方法论上交代:LLM 的何种输出特征可以被解读为「解释」?在哪些证据下这种解读成立、在哪些情境下不成立?Bubeck 等人以 GPT-4 为对象的早期实验显示这类系统在受控任务上表现出疑似「理解」的行为,但同样指出这些行为在分布外条件下迅速崩塌 [@bubeck2023sparks]。这一脆弱性意味着,把 LLM 放在 actor 位置的论文必须同时回答「在哪些条件下这一定位成立」与「在哪些条件下崩塌」这两个问题。

### 工程意义上的 agent 与本体论意义上的 agent

至此可以把两类 agent 概念分开。工程意义上的 agent 是第九章讨论的工作流自动化的延伸,它有目标、能分解任务、能调用工具、有失败模式可被监控;本体论意义上的 agent 是本章讨论的对象,它在研究叙事中占据真人 actor 的位置,论文论证的成立依赖于读者把它的输出当作真人会做的事情。

两种 agent 共享底层技术,论文的写作要求却并不重叠。工程 agent 的方法论披露,主要落在系统配置(模型版本、prompt 模板、工具集合、错误处理逻辑)和效果验证(任务完成率、错误模式分布)上;Ji 等人对生成系统幻觉的综述提供了这一类披露的标准 checklist [@ji2023hallucination]。本体论 agent 的方法论披露则要求额外回答两个问题:为什么这个 LLM 的输出可以代表真人 actor 的行为?哪些证据支撑这一代表关系?这两个问题在工程 agent 那里并不存在,因为没人会问「这段自动化脚本的行为是否可以代表真人行为」;它们只在本体论 agent 这一身份下成为论文方法部分必须回答的题目。

这一区分对实操具有直接含义。一个研究项目里同时使用了几台 LLM,有的用作脚本自动化(工程 agent),有的用作模拟被试(本体论 agent),论文的方法部分需要把它们分开描述,各自对应不同的披露标准与不同的 validation 要求;混在同一段叙述里,既会让评审无法判断方法论合规性,也会让复现者无从下手。Bommasani 等人对 foundation model 的整体性评估强调,模型用途的多样性必须在论文中被显式建档,否则一个看似单一的「论文使用 GPT-4 做了 X」陈述会同时遮蔽多种身份的方法论义务 [@bommasani2022opportunities]。


## 三种 LLM 出现在研究中的身份 {#sec-llm-three-identities}

### LLM-as-tool:工具身份的三个判定特征

**LLM-as-tool** 的特征是 AI 的存在不影响研究的因果叙事:把 AI 替换成「研究助理」「计算软件」「翻译工具」,论文论点不变。第四章给出过一个判断框架,把任务分为执行性与思考性两类,LLM-as-tool 完全落在执行性那一侧。前面九章讨论的工具用法,智能搜索、文献阅读、编程辅助、写作润色、图像生成,绝大多数都属于这一身份。

判定 LLM-as-tool 有三个方便的特征。其一,任务可以被独立地复核,即结果对错不依赖于 AI 本身的特性,而依赖于一个外部客观参照(数据库里有没有这篇论文、代码能不能跑通、英文表达是否符合期刊规范)。其二,作者的实质工作不会因为换一个 AI 工具而改变,论文论点对工具品牌中立。其三,论文方法部分不需要把 AI 描述为研究的一部分,只需要在使用披露段说明用了什么工具、用在哪个环节。Nature 在 2023 年初颁布的指南把这三条特征写成了具体政策语言,要求作者明确说明 AI 工具的使用范围,同时强调 AI 不应被列为作者 [@nature2023tools]。

值得特别提示的是,LLM-as-tool 这一身份并不豁免作者对内容的全责。Risko 与 Gilbert 关于「认知外包」(cognitive offloading)的综述显示,工具使用本身会改变使用者的判断模式 [@risko2016cognitive]。即便 LLM 只在一个执行性环节出场,只要其输出会反馈到作者后续的判断流程中,作者就有义务核对这一反馈是否扭曲了自己的推理路径;这条义务在工程意义上等同于代码审查,在方法论意义上等同于数据来源审计。第十三章关于域专家任务的三特征 [@sec-domain-expert-task-features] 已暗示这种反馈扭曲在专业判断场景下尤其难以被察觉,流畅性掩盖了正确性。

### LLM-as-agent:行动者身份的方法论义务

**LLM-as-agent** 把 AI 的位置向内推了一格:论文的论证不再围绕「研究者如何借助工具完成研究」,而是围绕「这个 LLM 在研究情境中如何行动」。Argyle 等人 2023 年用 GPT-3 模拟不同政治倾向美国选民对议题问卷的回答,Horton 把 LLM 当作 homo silicus 用于经济学行为实验,Aher 等人则把经典心理学实验在 LLM 上跑出与人类相近的反应模式 [@TODO-argyle-out-of-one-many-2023; @TODO-horton-llm-economic-agent-2023; @TODO-aher-using-llms-replicate-2023]。三组工作的共同特征是:研究的论点直接落在 LLM 的输出上,移除 LLM 的输出,研究本身不复存在。

这一身份要求论文方法部分把 LLM 写成研究的一部分,而不只是工具的一部分。具体落地为五项最低标准:模型与版本号、prompt 与角色设定模板、采样温度等关键超参数、跨多次重复抽样的稳定性指标、以及一组外部 validation 数据用以支撑「LLM 行为可代表人类行为」这一前提假设。前四项相对清晰,第五项是争议焦点:文献里关于 LLM 是否可以充当 silicon respondent 的整套讨论,实质上都在追问这一外部 validation 的可获得性 [@bommasani2022opportunities]。一个研究项目无法提供独立的真人对照数据时,所谓「LLM 模拟反映人类行为」的陈述就只是一个未经检验的方法学假说,而非可被引用的科学结论。

LLM-as-agent 的方法论义务也包含一类经常被低估的项目:跨模型版本的稳健性测试。LLM 服务方的模型权重持续变化,同一段 prompt 在 GPT-4o 与 Claude 3.5 上的输出分布可能差异显著,在 GPT-4o-2024-05 与 GPT-4o-2024-11 之间也未必稳定。Liang 等人在 HELM 框架下对多家服务方模型的横向比较显示,即使在标准化任务上,模型间差异已可达到决策性的量级 [@liang2023helm]。论文若仅在一台模型上跑硅样本即得出结论,后续读者无从判断该结论的稳健性,论文的科学价值随之坍塌。

### LLM-as-subject:研究对象身份的合法性

**LLM-as-subject** 是三种身份里方法论位置最清晰的一种:研究者直接以 LLM 自身为研究对象,研究 LLM 的能力边界、偏见结构、推理失败模式、对齐效果。Bubeck 等人对 GPT-4 涌现能力的早期实验、Dziri 等人对 Transformer 在组合推理上的限制研究、Huang 等人对自我纠错能力的实验、Bender 等人对大模型偏见与社会风险的批判,都属于这一身份 [@bubeck2023sparks; @dziri2024faith; @huang2024selfcorrect; @bender2021dangers]。Weidinger 等人的语言模型风险分类工作把 LLM-as-subject 研究的常见维度系统化为一份分类表 [@weidinger2022taxonomy]。

这一身份的合法性来自一个简单事实:研究的论点就是关于 LLM 的论点,不存在「这个发现是否可以推论到人类」的代表性问题。论文方法部分写法接近评测论文(benchmark / evaluation paper)的标准:模型清单、任务清单、评估指标、对照实验。Ouyang 等人关于 RLHF 的工作、Liang 等人关于 HELM 的工作,都为这一类研究提供了可被复现的实验配置范例 [@ouyang2022training; @liang2023helm]。

LLM-as-subject 的合法性也意味着对应的方法论严苛性。论文不能从「让 GPT 回答了 100 道题」直接跳到「LLM 具有 X 能力」,而需要在评测流程上回答几个标准问题:任务是否在训练数据中泄露、prompt 设计是否存在引导、采样是否对评估指标具有 robust 影响、跨模型对照是否揭示了能力差异而非工程差异。Marcus 在他长期对 AI 研究方法学的评论中反复指出,缺少这些控制的「能力声明」往往在后续复现尝试中被推翻 [@marcus2020next]。

### 身份混淆作为 AI 时代学术不端的新形态

以三身份为判别地图,可以看清当前文献中常见的一种新形态学术不端:用 LLM-as-tool 的轻率态度做了 LLM-as-agent 的工作,然后让结果的语义滑向 LLM-as-subject 都不会要求的「人类行为」语境。一篇典型的滑移论文会这样写:「研究使用 GPT-4 模拟了 500 个被试的道德判断,发现 X 现象;这表明人类在 X 情境下会有 Y 反应。」从工具部分开始的语言形式没问题,从结论部分开始的论点却不再有任何 validation 支撑,而中间的连接处刚好是论文叙事最容易被流畅性掩盖的位置。

这一类滑移与传统的数据造假或文献造假在形式上完全不同。它在每一步都是「合规」的(没造数据、没编引用、没改实验),累积起来的论文输出却让科学贡献变得空洞。第十三章关于域专家任务的三特征 [@sec-domain-expert-task-features] 已暗示这类问题的隐性:错误的代价高、错误的不可见性也高,流畅的科学叙述足以让读者(包括评审)在第一眼放过。Weidinger 等人对语言模型系统性风险的分类工作把这一类「沿叙事流形成的语义滑移」列为大模型在高风险领域应用时最难治理的风险类型之一 [@weidinger2022taxonomy]。这是 AI 时代学术不端最不容易被截获的一类,也是本章三身份判定法最直接的实操目标。

## 三身份判定法的快速测试

把任意一段「论文使用 LLM 做了 X」的论文叙述代入下面三步:

1. 把「使用 LLM」替换为「让一个研究助理」,论点是否仍然成立?如果完全成立,且论点不依赖 LLM 自身特性 → LLM-as-tool。
2. 把「使用 LLM」替换为「假设 LLM 行为可以代表 X 类人群」,论文是否还能为这一假设给出独立证据?如果论文论点的成立依赖这一假设 → LLM-as-agent。
3. 论点本身是否就是关于 LLM 的(LLM 在 X 任务上的能力 / 偏见 / 失败模式)?若是 → LLM-as-subject。

经过三步问答之后,论文应只落入一种身份。同一论点在两种身份之间滑移,即为前述「身份混淆」型学术不端的实证标志。判别工作不需要依赖外部专家,论文作者自己在投稿前完成一遍三步测试,就能截获绝大多数语义滑移问题。
:::


## 硅样本研究的方法论争议

第十一章把硅样本(silicon sample)作为方法论假设引入 [@sec-silicon-sample-as-method],并把它放在 RA Level 的边界上讨论。本章把这一争议拉回 Agent Level:一旦研究者声称硅样本可以代替真实样本,他实际上是在把 LLM 升格为研究中的 actor,接受 actor 身份附带的全部方法论义务。这一升格未必伴随论文叙述的形式变化,但伴随的是科学贡献的实质重定位。

### 乐观派的论证:训练数据中的人类痕迹

乐观派的论证建立在两条假设之上。第一条假设:LLM 在大规模人类文本上训练,内化了人类语言、态度、行为模式的统计结构。第二条假设:给定一个 demographic profile,LLM 在 prompt 条件下生成的回答,可以视为该 profile 在概率意义上的「典型回答」。Argyle 等人的 silicon sample 实验、Aher 等人将 Asch 从众实验、米尔格拉姆服从实验等经典范式在 LLM 上重跑的工作,都把这两条假设具体化为一组可被复制的实验流程 [@TODO-argyle-out-of-one-many-2023; @TODO-aher-using-llms-replicate-2023]。这些工作所提供的初步证据是:在若干被研究最广的议题(美国大选议题、主流消费偏好、经典道德困境)上,LLM 模拟的回答分布与真实调查数据具有相当一致性。

乐观派的应用主张相对克制。一类是预研:在大规模真实实验前,先用 LLM 生成假被试组,识别 prompt 与题目设计中的歧义、估计可能的样本量需求、做 power analysis,以降低真实实验的失败成本。另一类是不可行场景的替代:在某些 demographic 太难触及、伦理审查难以通过、成本过高的情况下,LLM 提供一种成本低廉的「估计版本」,使研究者能够在动用真实资源之前形成方向性判断。Park 等人在斯坦福搭建的「生成式 agent 小镇」即把这种应用推到了极端,用 25 个 LLM agent 模拟一个虚拟社区数周的日常互动,涌现出选举、聚会、流言传播等社会现象 [@TODO-park-generative-agents-2023]。从计算社会科学的方法学传统看,这一工作并不冒充真实社会,而是建立一个有方法论合法性的模拟系统。

乐观派的更深一层主张是:即使 LLM 模拟与真人行为存在偏差,这一偏差本身也具有研究价值。当一组主题在 LLM 上得到 X 分布、在真人上得到 Y 分布,X 与 Y 的差异结构可以提示训练数据中的覆盖偏差、文化偏差、议题热度偏差,从而成为社会语料学的研究对象。Bommasani 等人在 foundation model 的整体性评估里指出,这种「以 LLM 为透镜审视训练语料」的研究路径,本身就是一种对 LLM-as-subject 与 LLM-as-agent 的中间地带的合理利用 [@bommasani2022opportunities]。

### 怀疑派的论证:模拟与决策之间的鸿沟

怀疑派的反驳从一个简单观察开始:LLM 在 prompt 条件下生成的回答,反映的是训练数据中关于该 demographic 的人类描述,而不是该 demographic 的真实决策。这两件事并非总是同义。当被研究的问题在网络文本中被广泛讨论(如美国大选议题、主流消费偏好),模拟与真实可能高度一致;当问题新颖、文化特殊性强、或属于 demographic 不愿公开表达的私域(对家庭暴力的态度、对不熟悉政策的偏好、特定族群对宗教冲突的观点),模拟与真实可能严重偏离。

Bender 等人的「随机鹦鹉」一文给出的批判在这里特别相关:模仿语言的能力不等同于理解情境的能力 [@bender2021dangers]。怀疑派的进一步指控是:研究者常常无法事前判断自己面对的问题属于哪一类,因此用硅样本得到的「乐观结论」自带未知的偏差。Dillion 等人在 2023 年的一项评论里直接质问:把 ChatGPT 当作道德心理学的替代被试,在伦理学专业领域是否能产生任何可外推到真实人类的发现 [@TODO-dillion-chatgpt-moral-2023]。其论证主线是:LLM 的道德回答模式反映的是文本数据中关于道德判断的元描述,与人类在真实道德情境中体验到的情感张力、身体性、文化历史性维度在多个层面不同构。

怀疑派的另一条线索关注 LLM 输出的随机性结构。Dziri 等人对 Transformer 在组合推理任务上限制的研究表明,LLM 在需要稳定权衡的场景下表现出系统性失败模式 [@dziri2024faith];Huang 等人对自我纠错能力的实验则显示,LLM 在被要求修正自身判断时,常常陷入循环或漂移,无法稳定收敛到正确答案 [@huang2024selfcorrect]。这两类失败模式在硅样本研究中以隐蔽方式出现:研究者看到的是一组「看起来合理」的模拟数据,看不到的是这些数据在分布外条件下的脆弱性。Ji 等人对生成系统幻觉的系统综述把这一脆弱性总结为「流畅性掩盖事实性」的普遍模式 [@ji2023hallucination],而硅样本研究恰恰高度依赖这一被掩盖的事实性。

### 主流学术判断的四档划分

综合两派论证,2026 年初主流学术判断大致可分为四档,以「论文论点对硅样本的依赖程度」为主轴。

预研工具档:硅样本作为大规模真实实验之前的 pilot,用于探索可行性、识别问题、估计参数。这一档在多数主流期刊被允许,前提是论文清楚标注硅样本只是预研而非主分析,并在后续提供真人数据用于支撑论文主分析的论点。

主分析数据档:硅样本作为论文 main analysis 的样本来源,论文结论直接落在硅样本输出上并外推到人类。这一档目前几乎不被任何主流社科期刊接受,顶级心理学期刊的态度尤其严苛 [@TODO-dillion-chatgpt-moral-2023]。原因在于,缺少真人对照数据时,论文无法回答「这套结论何以适用于真人」这一最低质问。

研究 LLM 自身档:硅样本作为测量 LLM 偏见、行为模式、对齐效果的研究对象,完全合法,等同于上一节的 LLM-as-subject。这一档下,论文的科学贡献明确指向 LLM 自身,不存在向真人外推的方法论缺口。

身份混淆档:论文形式上声称是预研或 LLM-as-subject,实质上让结论滑入「人类如何如何」的语义,即为前述新形态学术不端。判别这一档不依赖外部技术工具,只要把论文论点拆开,逐句问「这一句的承担对象是 LLM 还是人类」,任何一句出现承担对象的偷换,论文即落入此档。

四档划分的实操含义在于:**硅样本能否作为方法被接受,并不取决于技术成熟度,而取决于研究者如何在论文中定位这套数据的来源。**同样一组 1000 条 GPT-4 输出,标注为预研可发,标注为主分析不可发,标注为 LLM 偏见测量完全合法。

### 硅样本研究的论文写作底线 {#sec-silicon-paper-rules}

把上述判断落地为论文写作的最低标准,硅样本研究有三条底线必须满足。

第一条:在引言或方法部分明确声明研究对象是 LLM 的行为模式,而非直接对应人类行为;若希望外推到人类,必须给出独立的 validation 数据(在同一情境下的真人对照实验,或一组已被验证的现实数据集)。声明的位置不应隐藏在论文末尾的局限性段落,而应出现在引言对研究问题的陈述之中,以便评审与读者从一开始就能正确解读论文论点的承担对象。

第二条:披露用于生成硅样本的全部工程细节。模型与版本号、prompt 与角色设定模板、采样温度与 top-p 等关键超参数、对同一 prompt 重复抽样的次数与稳定性指标。论文复现性的成立依赖于这些信息的完整披露;Ji 等人对生成系统幻觉的综述给出过类似披露要求的清单 [@ji2023hallucination]。一组未披露 prompt 模板的硅样本数据,等同于一组未披露问卷的人类调查数据,无法被其他研究者复核,也无法支撑论文论点的科学位置。

第三条:在讨论与结论部分严格限定推论的边界。结论只成立于该 LLM、该 prompt 模板、该类问题;不一定推广到其他模型、其他 prompt、或其他类型的问题。这一条与 Marcus 在更早的 AI 评论中关于 distribution shift 的警告呼应 [@marcus2020next],也对应第二章关于「锯齿状智能」的整章论证。论文若声称「这一发现具有跨模型的稳健性」,必须有跨模型重复实验的数据支撑;声称「这一发现可推广到其他议题」,必须有跨议题重复实验的数据支撑。无数据支撑的稳健性声明,在硅样本研究中尤其容易转化为科学过度延伸。


## 不同学科对 LLM-as-agent 的接受度差异

### 经济学:介于工具与代理之间的灰色地带

经济学对 LLM-as-agent 的接受度具有明显的层级差异。working paper 圈、应用经济学期刊、新兴的「行为经济学 + 大模型」交叉领域已经出现一批用 LLM 跑独裁者博弈、最后通牒博弈、谈判模拟、市场仿真的论文 [@TODO-horton-llm-economic-agent-2023]。这一类工作把 LLM 视为 homo silicus,一种 hypothetical economic agent,用于在低成本条件下生成大规模博弈结果,作为后续真人实验或政策仿真的方向性输入。

但 QJE、AER、Econometrica 等顶级期刊对这一方法保持距离。距离的来源不在于技术不成熟,而在于经济学方法论传统对「行为数据来源」的刚性要求:经济学实证传统建立在真人在真实激励下做出真实选择的数据之上;失去任何一个「真实」,论文的政策含义就被悬置。Horton 等人在自己论文里的处理方式,是把 LLM 输出明确定位为「behavioral hypothesis generation」(行为假设的生成),而非主分析数据 [@TODO-horton-llm-economic-agent-2023]。这一定位本身即承认了顶刊评审标准的现实约束。

经济学场景下还有一类应用值得单独提及:把 LLM 用于政策仿真。当一项政策的真实施行涉及复杂的多主体博弈,而真人实验在伦理或成本上不可行,研究者会用 LLM agent 群体模拟政策的潜在效应。这一应用接近第十六章将整合的「研究项目五阶段」中酝酿与构建阶段的功能,提供方向性洞察而非定量结论。Bommasani 等人对 foundation model 的应用前景评估指出,这种应用模式的合法性来自其对结论性质的克制,即它生成的是方向性洞察,而非可被引用的政策建议 [@bommasani2022opportunities]。

### 心理学:实证传统的方法学抵抗

心理学是对 LLM-as-agent 抵抗最强的学科。这一抵抗有方法论根源:心理学的实证传统建立在「真人被试 + 受控实验 + 重复测量 + IRB 审批」的链条上,任何环节缺位都会让研究的科学位置滑落。当 LLM 替代真人被试,这一链条的第一环节就被改写,后续所有环节(IRB 审批的对象、重复测量的统计假设、效应量的解释)都要重新回答,而目前心理学界没有形成关于这一系列重新回答的统一规范。

Dillion 等人 2023 年在 Trends in Cognitive Sciences 上的评论直接指出:把 ChatGPT 当作道德心理学的替代被试,既不能复制真人在道德困境中体验到的情感张力,也不能模拟道德决策中的身体性、文化性维度;研究 ChatGPT 在道德问题上的回答模式,只能告诉研究界 ChatGPT 自己在做什么,不能告诉研究界人类在做什么 [@TODO-dillion-chatgpt-moral-2023]。这一立场在 2024 至 2025 年间逐渐成为多数主流心理学期刊的隐性 default,顶级期刊的拒稿信中已开始出现「论文论点的承担对象不清」这一新型批评意见。

抵抗并非全面禁止。心理学界普遍接受 LLM-as-subject 类研究,即研究 LLM 自身在心理学经典任务上的表现模式,这是一项合法且有价值的工作 [@huang2024selfcorrect; @bubeck2023sparks]。区分点在于:论文论点的承担对象是 LLM 还是人类。承担对象一旦滑向人类,论文必须提供真人对照数据,否则评审会拒绝。心理学界的这一区分相对于经济学界更为严苛,部分原因在于心理学的研究产出直接进入临床、教育、公共健康政策的链条,误判的下游成本远高于经济学。

### 计算社会科学:LLM 作为新一代 agent-based modeling

计算社会科学是 LLM-as-agent 的天然栖息地。这一领域早就接受「用计算模拟代理真实社会过程」的方法学预设,即 agent-based modeling(ABM)在过去三十年里是该领域的标志性方法。LLM 只是为 ABM 提供了一种新的 agent 实现:更逼真的语言生成、更丰富的角色扮演、更接近真人的情境推理。从计算社会科学的视角看,LLM agent 不构成对真人样本的替代;它在方法学位置上对应于传统 ABM 中「过于简化的 agent」的升级形态,二者承担不同的科学位置。

Park 等人 2023 年在斯坦福搭建的「生成式 agent 小镇」是这一方向的标志性工作 [@TODO-park-generative-agents-2023]。25 个 LLM agent 在一个虚拟社区里持续互动数周,涌现出选举、聚会、流言传播等社会现象。从计算社会科学的角度看,这是一个有方法论合法性的研究,即它研究的是「在这套 agent 设定下,会涌现什么社会动力学」,而非「真实社会中也会有同样动力学」;论文论点的承担对象是模拟系统本身,而非真实社会。

合法性的边界仍然存在。计算社会科学界内部的辩论焦点是:LLM agent 的「逼真性」是否会让评审与读者错把模拟动力学当作真实社会规律。Bommasani 等人对 foundation model 的整体性评估指出,逼真度提升不会自动转化为外推有效性,反而可能放大「以假乱真」的风险 [@bommasani2022opportunities]。这一警告在计算社会科学的 LLM agent 研究中反复出现,促使该领域内部正在形成一组关于「模拟结论的标注规范」的共识,要求论文必须区分「在模拟中观察到的现象」与「该现象对真实社会的暗示」,前者是数据,后者是猜想,二者不能混在一段叙述里。

### 自然科学的 AI agent:不在同一维度的争议

自然科学中的 AI agent 与社科 agent 在方法论上几乎不在同一维度。化学合成 agent(给定目标分子,AI 自主规划合成路径并控制实验)、材料发现 agent(自动跑大量虚拟材料模拟,挑选有前景的候选)、药物筛选 agent(给定靶点,AI 自动设计候选分子),都属于这一类。Boiko 等人 2023 年在 Nature 发表的自主化学合成 agent 是这一方向的代表性工作 [@TODO-boiko-autonomous-chemistry-2023];这一系统能根据目标分子自主规划反应路径,并通过控制实验仪器完成实际合成。

自然科学 AI agent 的方法论争议远小于社科 agent,有三个原因。其一,结果有客观验证:合成出来或者没合成出来、材料表现出预测的性能或者没有、药物结合靶点或者不结合;agent 的行为不需要被解释为「真人会做的」,只需要在客观参照下评判。其二,agent 不冒充人类被试,它是高度自动化的工具,不进入「actor 是否能代表真人」的方法论争议;论文方法部分的写法接近实验报告而非社会科学论文。其三,可复现性强:同样的输入参数加同样的模型版本,基本能复现结果,随机性主要来自实验本身而非 agent 行为。

自然科学 AI agent 与第七章的编程辅助、第九章的工作流自动化在概念上是连续的,它们是 AI 工具的极端自动化形态,而非「AI 替代研究 actor」的本体论转变。它们引发的问题集中在治理层面:自动化研究的速度爆炸如何被同行评议跟上、自动化生成的 candidate 谁来验证、生成与验证之间的速度差是否会让「未被验证的发现」涌入文献库。这些问题与本体论意义上的 actor 概念无关,而是组织与制度问题,将在第十五章治理章节展开。Weidinger 等人的 LLM 风险分类工作把这一类「自动化产出超过验证能力」列为大模型在科学研究中应用时最需要治理框架的风险类别之一 [@weidinger2022taxonomy]。


## 「行动者」概念的边界 {#sec-actor-concept-boundary}

### actor 的三个传统条件

社会科学传统对 actor 的最低承诺,可以总结为三个条件。**第一个条件是面对相同情境时具有不同选择的能力**,这是「自由意志」概念在科学语境下的弱化版本,亦即不要求形而上意义的自由,只要求行为不被情境完全决定。**第二个条件是具有可被独立指认的内部状态**:信念、偏好、记忆、目标,这些状态在不同时刻具有连续性,在跨情境中可以被一致地推断。**第三个条件是对自身行动具有某种程度的归属感**,即责任的雏形,行动者能在事后被追问「为什么这样做」并给出基于自身视角的回答。

这三个条件并不要求被研究的 actor 是人类。机构、组织、动物在不同传统下都可以满足部分条件,因而被纳入相应学科的 actor 概念。但是,放弃任何一个条件都会让相应的方法论范式部分失效:没有第一个条件的系统,博弈论的 best response 概念无意义;没有第二个条件的系统,纵向研究的 panel data 假设无意义;没有第三个条件的系统,任何涉及问责的政策设计无意义。这一可加可减的条件结构提示一个判别策略:LLM 在三个条件上的表现需要被逐条审视,而不是被整体上的「像不像 actor」的直觉判断所替代。

### LLM 在三个条件上的表现

在第一个条件「面对相同情境的不同选择」上,LLM 在 prompt 完全相同的情况下,确实因采样温度而产生不同输出。但这一不同是温度参数控制下的统计抽样,而非一个 actor 在权衡后做出不同选择的过程。Dziri 等人对 Transformer 在组合推理任务上限制的研究表明,LLM 在需要稳定权衡的场景中表现出系统性失败模式,与「actor 进行权衡」的过程在机理上不同构 [@dziri2024faith]。从外部观察看,LLM 输出与 actor 选择都是「同一情境下的不同行为」;从内部机制看,前者是采样,后者是权衡,二者不能在方法论上等同。

在第二个条件「内部状态的连续性」上,LLM 没有跨会话的内部状态,即每个新对话从一个空白上下文开始,模型权重之外不保留任何与具体被试历史绑定的信息。即便在长上下文窗口内,被记录的也只是输入文本而非「内化的信念或偏好」。Kahneman 在 System 1 与 System 2 框架下强调的人类决策的迭代修正过程,在 LLM 这里没有对应物 [@kahneman2011thinking];Risko 与 Gilbert 关于认知外包的研究也指出,人类决策的连续性本身依赖于跨情境的记忆与反馈机制,这一机制在 LLM 的会话架构里几乎完全不存在 [@risko2016cognitive]。

在第三个条件「行动归属感」上,LLM 不能为自己的输出承担后果,亦即它没有 stake、没有声誉、没有未来情境会因为现在的选择而改变。Marcus 关于稳健 AI 的早期评论指出,这种「无后果性」是当前 LLM 与传统 actor 概念之间最难弥合的差距 [@marcus2020next]。在博弈论意义上,LLM 不是一个能在重复博弈中累积声誉、能因背叛而承担长期成本的 player;在社会学意义上,LLM 不是一个嵌入于关系网络、能因行动而获得或失去 status 的 member;在伦理学意义上,LLM 不是一个能被问责的 agent。三个意义上的「不是」叠加,使得把 LLM 直接放进 actor 位置的研究在方法论上承担了一层未被声明的额外义务:解释为什么在这些「不是」的条件下,LLM 的行为仍然可以代表 actor 在严格意义上的行为。

按这三条评估,LLM 不属于传统意义上的 actor;它构成一个能模拟 actor 行为的统计系统。这一定位不带贬义,只是对方法论身份的精确说明,以便后续的研究设计、论文写作、同行评议都建立在正确的概念基础上。

### 「能模拟 actor 行为的统计系统」的科学贡献边界

把 LLM 定位为「能模拟 actor 行为的统计系统」之后,这一类研究允许什么、禁止什么,就有了清晰的边界。

允许的工作包括:研究该 LLM 在某类任务上的行为分布、研究 LLM 行为对 prompt 设计的敏感性、研究不同模型版本之间的行为差异、研究模拟行为与真实人类数据的偏差结构。这些工作的科学贡献都明确指向 LLM 自身,对 AI 评测、对齐研究、模型审计具有直接价值。Bommasani 等人对 foundation model 的整体性评估、Liang 等人的 HELM 框架,都属于这一允许范围内的范例 [@bommasani2022opportunities; @liang2023helm]。

禁止的工作主要是一类:把 LLM 模拟得到的行为模式直接外推为真人行为规律,且不提供独立的真人 validation 数据。这类研究即使从形式上看符合「数据 + 分析 + 结论」的科学叙事,实质上也无法回答「这套结论何以适用于真人」的最低质问。它们的科学贡献,严格说,只对该特定 LLM 在该特定 prompt 下的输出特性有意义,而不对一般博弈理论、心理学规律、社会过程理解有意义。论文若希望突破这一边界,唯一被方法论认可的路径是引入真人对照数据,把硅样本研究升级为「硅样本 + 真人样本」的组合研究,并在论文中明确分析两套数据的一致性与偏差。

### 边界混淆的两类典型科学过度延伸

边界混淆在文献中以两种典型形态出现。

第一种是用 LLM 谈判结果推论真人谈判规律。研究者用 LLM 跑出 1000 次最后通牒博弈,观察出价分布,声称这一分布反映了人类在最后通牒博弈中的策略偏好。问题是:LLM 的出价分布反映的是其训练数据中关于最后通牒博弈讨论的统计结构,可能与现代实验经济学积累的真人数据高度一致(因为这一议题在文献里被广泛讨论),也可能在某些条件下严重偏离(例如非西方文化下的博弈实验,或在未被广泛讨论的新型博弈结构上)。论文若不附真人对照,无法分辨自己面对的是哪一种情况;读者也无从判断结论的外推有效性。

第二种是用 LLM 道德判断推论人类道德。研究者用 LLM 在数十个伦理两难问题上生成回答,统计出「LLM 倾向于某类回答」的模式,声称这一模式反映了人类道德直觉的某种结构。Dillion 等人在评论里直指这一类研究的方法论缺陷:LLM 的道德回答模式反映的是文本数据中关于道德的元描述,与人类在真实道德情境中的体验性判断在多个维度上不同构 [@TODO-dillion-chatgpt-moral-2023]。第十三章关于域专家任务的三特征 [@sec-domain-expert-task-features] 已暗示这种不同构:领域判断需要内化的隐性知识,LLM 在这类任务上的失败模式具有锯齿状分布,流畅性掩盖了正确性,而道德判断恰恰是「域专家任务」的极端情形,既需要内化的伦理直觉,也需要对情境的细腻把握,二者都不是统计采样能直接覆盖的。

两类过度延伸的形态不同,根源相同:论文论点的承担对象在叙述过程中发生了无声的偷换,而读者(包括评审)难以从论文的形式特征上识别这一偷换。三身份判定法、四档划分、三条写作底线,正是为了在这一无声偷换发生之前,把判别工具放在研究者自己的手里。

::: {.callout-warning}
## 硅样本研究的论文写作三项最低披露

凡涉及 LLM 输出被用于代表人类行为或被推论为一般规律的研究,论文方法部分应满足以下三项最低披露,否则即落入前文所述的身份混淆型学术不端。

1. 明确论点的承担对象。声明论文研究的是 LLM 行为模式还是人类行为模式;若是后者,提供独立的真人 validation 数据(同情境下的真人对照实验或已被验证的现实数据集)。
2. 完整披露生成硅样本的工程细节。模型与版本、prompt 与角色设定、采样温度与 top-p、跨重复抽样的稳定性指标。复现性的成立依赖于这些信息。
3. 严格限定推论边界。结论只成立于该模型、该 prompt、该类问题;不假定可推广至其他模型、其他 prompt、其他问题类型。

这三项并非简单的合规清单。它们构成把 LLM-as-agent 研究做成科学而非流行的最低门槛。一个研究者若无法写出完整的三项披露,其研究的科学贡献只能是「这个特定 LLM 在这种情况下产出了这些 token」;这一陈述与论文叙事承诺给读者的科学发现之间存在一道难以跨越的鸿沟。
:::


## 论证性收束:本体论的边界从来不是技术问题

本章的论证可以归结为三个相互嵌套的命题。第一,Agent Level 与前三级角色的差异不是程度差异,而是本体论差异,即 AI 从研究的外部位置进入研究的内部位置,论文叙事的承诺对象由此发生改写。第二,三身份判定法(LLM-as-tool / LLM-as-agent / LLM-as-subject)是研究者识别这一改写的实操工具,身份混淆是 AI 时代学术不端最隐蔽的新形态,它既不伪造数据也不编造引用,却让论文的科学贡献在形式合规的叙述下逐步被掏空。第三,「行动者」概念的边界讨论指出 LLM 的方法论身份是「能模拟 actor 行为的统计系统」,这一定位允许严格的 LLM-as-subject 研究,允许有 validation 的 LLM-as-agent 研究,但不允许把 LLM 模拟直接外推为真人规律。

三个命题在全书脉络中处于一个特定位置。第一章已给出 Agent 在工程意义上的定义,把 Agent 描述为「你给它一个目标而不是一个问题」的系统;第四章关于执行性与思考性任务的边界(line 130-136)已为本章的本体论扩张埋下伏笔,指出当自动化系统的输出被放入研究者本应承担的位置时执行与思考的边界会被踩过;第十一章把硅样本作为方法论假设引入 [@sec-silicon-sample-as-method],第十三章关于域专家任务的三特征 [@sec-domain-expert-task-features] 提示了 AI 在专业判断上的失败模式。本章在前述伏笔之上,把「AI 进入研究本体」这一最深的方法论问题展开为可操作的判别框架,使研究者能在论文写作的早期就识别身份滑移的潜在路径。第十五章将进一步追问:当三身份判定法与硅样本写作底线都依赖于个体研究者的自律,而个体自律在大规模 AI 论文产出面前节节失守时,学术体制本身需要承担哪些新责任。

本章的论证留下若干开放问题,值得在此明确承认而非掩盖。其一,LLM 是否会在能力进一步提升后,在严格意义上满足传统 actor 概念的三个条件?如果答案在某种意义上为肯(例如未来某代模型具有跨会话持久内部状态、具有可被外部约束的行动后果),本章给出的判别框架需要重新评估;届时硅样本与真人样本的方法论区分本身可能需要重写。其二,不同学科对 LLM-as-agent 的接受度差异(经济学保留、心理学抵抗、计算社会科学接纳),会随时间收敛到某个共识,还是会因为各学科本体论假设的差异而长期分化?这一问题没有先验答案,只能由各学科在未来五到十年的实证经验回答;本章对四学科的描述也只是 2026 年初这一时间切片上的快照。其三,本章对 LLM-as-agent 的判别主要集中在社会科学语境;化学、材料、药物筛选等自然科学 AI agent 的方法论问题,与社科 agent 共享技术底层但在哲学预设上几乎不重叠,值得另起一套独立的方法论讨论框架,这一框架的建立超出本书范围,留待自然科学方法学者承担。

研究者在自己的工作中如何处理这一系列开放问题,既取决于 LLM 自身的发展速度,也取决于学术共同体在治理层面如何回应。本章把判别工具放在研究者手里,使其能在论文写作的早期就识别身份滑移的潜在路径;第十五章把同样的问题升到学术体制的层面,追问当个体判别工具的边际效用递减、当 AI 论文产出的速度远超同行评议的承载力,学术体制需要承担什么样的新责任。Agent Level 的本体论扩张到此告一段落,Governance Level 的体制讨论由此承接。

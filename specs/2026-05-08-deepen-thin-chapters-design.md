## 1. 背景与目标

### 1.1 现状诊断

AAAR 全书 18 章已全部产出草稿,但深度分布严重不均:

| 范围 | 章节 | 字节数(中位) | TODO 占位 | 状态 |
|---|---|---|---|---|
| 早期产出 | 01-04 | 24K-47K | 0 | 体量充足 |

| **后段产出** | **07-18** | **9K-16K** | 1-6 | **体量约为基线的 1/3** |

进一步样本核查显示,体量缺口背后是结构性深度缺口:

- ch04 (36.9K) 嵌入 5+ 篇真实学术研究(Slamecka & Graf 1978、Bjork 1994、Kahneman、Flower & Hayes 1981、Carr、Sparrow et al.),平均每个二级节 2-3 个具体例子;

### 1.2 一个关键修正

在前期分析中曾把 ch01-04 视为"作者人写基线"。**该假设不成立**——根据作者本人确认,ch01-04 同样是 AI 起草。因此:

- 已有的 `VOICE.md` 是某次模型输出的统计画像,不是作者真实写作声音;
- 不能以 ch04 的频次数据作为"可接受 AI tell 上限"的辩护;
- 新章节的目标不是"看起来像 ch01-04 的延续",而是"读起来像一本通过同行评审的中文学术著作的论述部分"。

### 1.3 目标

把 ch07-18 共 12 章扩写到深度可用的状态,具体包括:

1. **体量目标**:每章 ≥ ch04 当前体量(35K+),part 3 核心章节(11-15)力争 40K+;
2. **学术调性**:从对话式叙述转为传统学术论述体,系统性剥离常见 AI 文本特征;
3. **学术密度**:每章 ≥ 8 条真实学术引用 + 每个二级节 ≥ 2 个具体研究案例(可含 TODO 占位待人补充经验性案例);
4. **结构成熟度**:每章 4-6 个二级节,每个二级节内含 2-4 个三级节;
5. **跨章一致**:章际交叉引用准确;术语前后一致。

### 1.4 非目标(显式排除)

- **不修订 ch01-06**:作者已锁定范围 07-18;ch01-06 调性虽与新章不同,作为"对话式入门"可与"学术论述主体"形成自然分层,不强行抹平;
- **不重新发明 sub-agent / skill**:复用项目已有 `chapter-author` + `chapter-reviewer` agent + `chapter-template` / `applying-voice` / `finding-citations` / `checking-coherence` skill;
- **不在 spec 阶段写细纲**:每章二级节具体安排留给 author agent 在 ideas + 模板约束下展开,本 spec 只锁结构骨架与质量门;
- **不替作者注入个人经验**:涉及作者亲身案例的部分继续以 `<!-- TODO: 作者补充 -->` 占位待人填补;
- **不修订 references.bib 之外的引用**:严禁伪造任何 BibTeX 条目。

### 1.5 与 2026-05-07 spec 的关系

2026-05-07 spec 明确写过"不做并行多章节生成(顺序生成才能让后写章节看到前面的上下文)"。本 spec 显式推翻该决定,理由:

1. 当时仓库还没有"锁包"机制,顺序生成是唯一保证一致性的手段;
2. 本 spec 引入的 5 件锁包(VOICE-academic / blocklist / template / citation policy / cross-reference contract)在 sub-agent 启动前已固定,跨章节一致性的关键决策点在并行启动时已经全部确定;
3. 12 章顺序生成的耗时不可接受,且每章顺序生成对前章的上下文实际依赖被 cross-reference contract 显式化后变得有限。

## 3. 锁包详细规格

### 3.1 VOICE-academic.md

**位置**:仓库根。文件名固定为 `VOICE-academic.md`(与现有 `VOICE.md` 并存,不替换;供 ch07-18 author agent 强制读取)。

**核心规则**(每条均需在文件中给 1-2 个反例 + 改写示范):

| 维度 | 旧 VOICE.md(对话体) | 新 VOICE-academic.md(学术体) |
|---|---|---|
| 主语人称 | "你"高频(ch04 = 241 次/章) | 第三人称("研究者""学者""作者""读者")为主;"你" ≤ 20/章;严禁"我们"召唤("让我们……"禁) |
| 句长 | 长短交替,关键转折用短句锤定 | 以长复合句为主,通过从句、并列结构展开论证;短句仅在结论性命题处使用 |
| 破折号 —— | 极常用(ch04 = 85 次/章) | ≤ 5/章;仅用于真正的插入语中断;并列扩展用顿号、解释关系用"即/也就是"或独立句 |
| 加粗 `**...**` | 每节 1-2 处锚定(ch04 = 68 次/章) | ≤ 10/章;仅用于首次出现的关键术语 + 全章 ≤ 3 个核心论断锚定 |
| 类比 | 身体/餐厅/工业等多类高频 | 慎用;若用应直接服务于专业概念解释,不堆叠 |
| Callout | 每章 4-7 个 | ≤ 2/章;callout 标题改名词短语(例:"认知外包的隐性成本"而非"警惕认知外包") |
| 章末 | 原则清单 + 一句引下章 | 论证性收束:把本章核心命题与全书脉络的关系说清;不用列表清单 |
| 章首 | 年份+真实事件 / 上一章回顾 / 假设场景 | 改用研究问题陈述、经验观察陈述、或定义性引入;不再用"年份+故事"开篇 |

**学术密度目标**(在文件中作为硬性最小值):

- 每章真实学术引用 ≥ 8(已存在于 references.bib 的 key)
- 每个二级节具体案例 ≥ 2(真实研究 / 真实事件 / 具体场景三选一)
- 每章三级节(`### 标题`)总数 ≥ 8

**强制读取**:文件 frontmatter 显式声明"本文件适用于 ch07-18,优先于 VOICE.md"。

### 3.2 AI-tells-blocklist.md

**位置**:仓库根。

**18 条禁忌**(数据驱动 8 + 经验补充 10),每条结构化为:`{模式名}` / `{反例(典型 AI 句)}` / `{改写后示范}` / `{每章上限}`:

数据驱动 8 条:

1. 破折号 —— 滥用 — 上限 5/章
2. 加粗 `**...**` 滥用 — 上限 10/章
3. "你"频率过高 — 上限 20/章
4. "让我们"召唤式 — 禁
5. "A 不是 X 而是 Y"对仗 — 上限 1/章
6. "核心 / 关键 / 本质 / 根本 / 真正"空抽象 — 上限合计 5/章
7. "这意味着 / 换句话说 / 值得注意的是 / 事实上 / 实际上"元评论 — 上限合计 1/章
8. "就像 X 一样"整齐比喻 — 上限 1/章

经验补充 10 条:

9. 中英对照术语过密("嵌入(embedding)、注意力(attention)、上下文(context)"三词联缀)— 同一术语首次出现给 1 次中英对照,后续单用
10. "需要指出的是 / 应当注意 / 可以认为 / 不难看出"学术腔标配 — 上限 2/章
11. "在 X 时 / 当 X 时"时间状语前置 — 上限 5/章;改后置或嵌入
12. "首先 / 其次 / 再次 / 最后"枚举式过渡 — 禁
13. "总而言之 / 综上所述 / 简而言之"封闭式总结 — 章节内禁;章末仅允许 1 次
14. 节末加粗"金句"收尾(每节末用 `**…**` 升华一句)— 禁
15. "这正是 X 想说的""这正是 X 的意义所在"自指元话 — 禁
16. "真正的 X 在于 / 问题的关键是"公式化转折 — 禁
17. 整齐 N 项枚举(常见 3 / 5 项)— 真要枚举允许参差,禁为对称凑数
18. callout 标题用"警惕 X / 注意 X"祈使式 — 改名词短语

**强制度**:每条都是硬约束。reviewer agent 必须逐条扫描并在 REVIEW-chXX.md 中报告每条违规处。超过任一上限的章节必须返工。

### 3.4 citation-policy.md

**位置**:仓库根。

**强约束**:

1. 引用语法只能是 `[@key]`,key 必须存在于 references.bib(可由 finding-citations skill 验证);
2. 找不到合适已有 key 时,使用 `[@TODO-描述性 slug]`(例:`[@TODO-llm-peer-review-2024]`),严禁伪造任何条目;

## 4. 三波并行执行流

### 4.1 总体流程

```
Phase 0(主线顺序)
  ├── 写 VOICE-academic.md
  ├── 写 AI-tells-blocklist.md
  ├── 写 chapter-template-academic.md
  ├── 写 citation-policy.md
  ├── 写 cross-reference-contract.md(基于阅读 ideas/ + 现有薄稿)
  └── git commit 锁包

Phase 1 ── 4 个 chapter-author sub-agent 并行(ch07/08/09/10)
  ├── 每个 author 读:ideas/XX + 锁包 5 件 + 既有薄稿(参考其结构与 TODO 占位,不照抄)
  ├── 每个 author 写完后同 phase 内自动触发 chapter-reviewer
  ├── 主线 Claude 等所有 4 个完成后:汇总 4 份 REVIEW-chXX.md;跑 checking-coherence
  ├── 致命问题(blocklist 严重违规 / 引用伪造嫌疑 / 跨章引用断裂)→ 返工那一章
  └── 用户抽样验收 1-2 章 → 全过/微调锁包后过/拒收
        └── 拒收 → 锁包大改 → 整波重做

Phase 2 ── 5 个并行(ch11/12/13/14/15) — 同上结构
Phase 3 ── 3 个并行(ch16/17/18) — 同上结构

Final pass
  ├── checking-coherence 全书扫
  ├── grep 全 12 章 blocklist 残留
  └── 主线 Claude 报告 + 用户最终签字
```

### 4.2 sub-agent 调用契约

每个 chapter-author sub-agent 的输入:

```
任务:把 outline/XX-*.md 重写为深度学术论述,目标 ≥ 35K(part 3 核心章 ≥ 40K)。
强制读取(按顺序):
  1. ideas/XX-*.md(原始素材与作者意图)
  2. VOICE-academic.md(调性硬约束)
  3. AI-tells-blocklist.md(18 条禁忌)
  4. chapter-template-academic.md(结构骨架)
  5. citation-policy.md(引用政策)
  6. cross-reference-contract.md(交叉引用契约)
  7. outline/XX-*.md(既有薄稿,作为意图参考,不照抄结构)
  8. 同 phase 已完成章节(若有)+ ch01-06(供承上)

产出:
  - 覆写 outline/XX-*.md
  - 不创建任何新文件
  - 涉及作者亲身案例处保留 <!-- TODO: 作者补充 --> 占位
  - 引用未找到 key 时使用 [@TODO-slug] 占位
  - 写完后自动触发 chapter-reviewer sub-agent
```

每个 chapter-reviewer sub-agent 的输入与产出:

```
任务:复核新写的 outline/XX-*.md 是否符合锁包要求,产出 REVIEW-chXX.md。

输出文件结构:
  - § A 字数与结构合规(章字数、二级节数、三级节数、callout 数)
  - § B Blocklist 18 条逐条扫描(每条违规处行号 + 原文摘录)
  - § C 引用政策合规(真实引用数、TODO 数、有无伪造嫌疑)
  - § D 交叉引用契约合规(锚点 ID 引用是否存在、有无前向引用违规)
  - § E 学术密度合规(每节具体案例数)
  - § F 综合判定:PASS / NEEDS_REWRITE / NEEDS_TARGETED_FIX

reviewer 严禁修改 outline/XX-*.md,只产出 REVIEW-chXX.md。
```

### 4.3 验收门定义

每个 phase 完成后,主线 Claude 必须在进入下一 phase 前完成:

1. 读完该 phase 所有 REVIEW-chXX.md;
2. 对每章给出"该重跑 / 局部修补 / 通过"判定;
3. 将所有 PASS 章节做一次跨章一致性扫(术语 / 锚点 / 互相引用);
4. 把 phase 总结(每章字数、引用数、blocklist 违规数、跨章问题清单)汇报给用户;
5. 等用户签字"全过"或给出"微调锁包后过"的具体调整指令;
6. 若用户给"拒收 + 重做",回到锁包修订并标记该 phase 状态为待重启。

### 4.4 失败处理

| 失败模式 | 处理 |
|---|---|
| 单章 reviewer 判 NEEDS_REWRITE | 主线立即在该 phase 内重启该章 author + reviewer |
| 单章 reviewer 判 NEEDS_TARGETED_FIX | 主线作小补丁(grep + 局部 edit),不动其他章 |
| 同 phase 多章一致性不齐(术语 / 调性) | 整波回到 Phase 0 微调锁包 → 整波重做 |
| 用户验收发现锁包遗漏(新一类 AI tell / 新约束) | 锁包补条目 → 已完成章用 reviewer 重扫 → 局部修补;不重写整章 |
| 全部完成后 final coherence 发现跨章漂移 | 局部修补;若漂移严重则 Phase 3 重做 |

## 5. 风险

| 风险 | 严重度 | 缓解 |
|---|---|---|
| 锁包不够具体 → sub-agent 仍出 AI 味稿 | 高 | Phase 1 是早期暴露 → 把 Phase 1 当锁包验证 phase,4 章是最小可信样本 |
| BibTeX 引用伪造 | 中 | citation-policy 强制 + reviewer 必扫 + finding-citations skill 双重校验 |
| 跨章交叉引用断裂 | 中 | cross-reference-contract 在并行启动前固定 + final pass coherence 兜底 |
| 用户验收负担过重(12 × 35K ≈ 420K 中文) | 高 | 每 phase 只要求抽样 1-2 章读全;reviewer 报告作为"是否值得读全"的 triage |

| Quarto 渲染失败(语法错误) | 低 | 每章 author 完成后跑 `quarto check`;reviewer 报告附带渲染状态 |
